# AI Dev Context - TubeShelf

このファイルは、本プロジェクト「TubeShelf」の開発を支援するAIエージェント（あなた）のためのコンテキスト情報です。
プロジェクトの概要、技術スタック、開発ルールを理解し、効率的かつ整合性の取れたコード生成・提案を行ってください。

## 1. プロジェクト概要・要件
プロジェクトの詳細な概要、背景、および機能要件については、以下の要件定義書を参照してください。

- **要件定義トップ**: [`docs/00_requirements/README.md`](docs/00_requirements/README.md)
    - システム要件、機能要件、非機能要件などが定義されています。

詳細な機能仕様や設計については、`docs/` 以下の各フェーズのドキュメント（基本設計、詳細設計など）を順次参照してください。

## 2. 技術スタック・環境構築
技術スタックおよび環境構築の詳細については、以下のドキュメントを参照してください。

- **開発環境・技術スタック**: [`docs/30_development/build_setup.md`](docs/30_development/build_setup.md)

## 3. ディレクトリ構成と開発プロセス
本プロジェクトは、ウォーターフォールモデルを意識したディレクトリ構成でドキュメントを管理しています。

- `docs/`
    - `00_requirements/`: 要件定義
    - `10_basic_design/`: 基本設計（アーキテクチャ、データモデル）
    - `20_detail_design/`: 詳細設計（コンポーネント、内部ロジック）
    - `30_development/`: 開発規約、環境構築
    - `40_unit_test/`: 単体テスト
    - `50_integration_test/`: 結合テスト
    - `60_deployment/`: デプロイ、リリース

### AIへの指示ルール
1.  **ドキュメントファースト**: 実装を行う前に、関連する `docs/` 以下のドキュメントを確認、または更新を提案してください。
2.  **整合性の維持**: 既存の設計（`docs`内の記述）と矛盾するコードを生成しないでください。変更が必要な場合は、設計の修正から提案してください。
3.  **言語**: ドキュメントおよびコミュニケーションは基本的に **日本語** を使用してください。コード内のコメントも **日本語** で記述してください。変数名・関数名等は英語を使用してください。
4.  **質問への回答**: ユーザーからの質問には可能な限り以下の観点を含めて回答し、意思決定を支援してください。
    - **基本的な回答**: 結論や直接的な答え
    - **メリット・デメリット**: 推奨するアプローチの利点と欠点
    - **代替案**: 他の選択肢とその特徴
    - **その他の情報**: 補足、関連するコンテキスト
