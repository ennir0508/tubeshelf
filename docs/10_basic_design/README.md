# 基本設計 (Basic Design)

このディレクトリでは、要件定義に基づき、システムの全体構造、機能配置、データ構造の基本方針を策定した成果物を管理します。
詳細設計および実装のスムーズな進行を目的として、アーキテクチャやデータモデルを定義します。

## ドキュメント構成

本ディレクトリには、以下のドキュメントを配置します。

| ドキュメント                   | ファイル名                                       | 内容概要                                                                                       |
| :----------------------------- | :----------------------------------------------- | :--------------------------------------------------------------------------------------------- |
| **アーキテクチャ設計書**       | [`architecture.md`](./architecture.md)           | システム全体の構成図、コンポーネント間の連携、採用技術の選定理由などを記述します。             |
| **画面遷移設計書**             | [`screen_transition.md`](./screen_transition.md) | 画面間の遷移フロー、状態遷移の詳細を定義します。UI要件定義をベースに実装レベルへ具体化します。 |
| **データモデル設計書**         | [`data_schema.md`](./data_schema.md)             | Chrome Storageおよび内部ステートで保持するデータのスキーマ定義、型定義を行います。             |
| **API/インターフェース設計書** | [`interface_design.md`](./interface_design.md)   | モジュール間や外部API（YouTube Data API等）とのインターフェース仕様を定義します。              |
