# 要件定義 (Requirements Definition)

このディレクトリでは、TubeShelfプロジェクトの要件定義工程における成果物を管理します。
ウォーターフォールモデルに基づき、システム要件から機能・非機能要件、UI/UX、データ要件へと詳細化を行い、設計工程へのインプットを作成します。

## ドキュメント構成

本ディレクトリには、以下のドキュメントを配置します。

| ドキュメント           | ファイル名                             | 内容概要                                                                                                                         |
| :--------------------- | :------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------- |
| **システム要件定義書** | [`sys_req.md`](./sys_req.md)           | プロジェクトの背景、目的、システム化の範囲、想定ユーザー（ペルソナ）、全体的な業務フローなどを定義します。                       |
| **機能要件定義書**     | [`func_req.md`](./func_req.md)         | システムが実装すべき機能の一覧と詳細を定義します。（例：登録チャンネルのカテゴリ化、サイドバー表示、ドラッグ＆ドロップ操作など） |
| **非機能要件定義書**   | [`non_func_req.md`](./non_func_req.md) | パフォーマンス、セキュリティ、可用性、保守性などの品質要件や、ブラウザ互換性などの環境要件を定義します。                         |
| **画面設計書 (UI/UX)** | [`ui_req.md`](./ui_req.md)             | 画面レイアウト、画面遷移図、各画面の構成要素や操作時の振る舞いを定義します。                                                     |
| **データ要件定義書**   | [`data_req.md`](./data_req.md)         | アプリケーションで扱うデータ構造、Chrome Storageへの保存形式、リレーションなどを定義します。                                     |
