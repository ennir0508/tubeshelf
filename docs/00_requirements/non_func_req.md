# 非機能要件定義書 (Non-Functional Requirements Definition)

## 1. 性能要件 (Performance)

### 1.1. レスポンスタイム
- **サイドバー表示**: YouTubeのページロード完了後、1秒以内にカスタムサイドバーが表示されること。
- **ドラッグ＆ドロップ**: 操作時の追従遅延が人間が感知できないレベル（16ms以下推奨）であること。ドロップ後の反映は即座に行われること。

### 1.2. リソース消費
- **メモリ使用量**: 拡張機能による追加メモリ消費は最小限に抑え、YouTube動画再生のパフォーマンスに悪影響を与えないこと。

## 2. セキュリティ要件 (Security)

### 2.1. データ保護
- ユーザーのカテゴリ設定データは、Chrome Storage (Local/Sync) 内に安全に保存する。
- 外部サーバーへのデータ送信は行わない（プライバシー重視）。

### 2.2. DOM操作の安全性
- YouTubeの正規のスクリプトや他の拡張機能と競合しないよう、固有のIDやClassプレフィックスを使用し、影響範囲を限定する。

## 3. 可用性・信頼性 (Availability & Reliability)

### 3.1. オフライン動作
- インターネット接続がない状態でも、キャッシュされた設定に基づきサイドバーが表示されること（動画再生自体は不可でも、メニュー構成は維持）。

### 3.2. 同期 (Sync)
- Chrome Sync Storageを利用し、同じGoogleアカウントでログインしている複数のPC間でカテゴリ設定が同期されること（容量制限に注意し、必要ならLocalへ切り替えまたは圧縮を検討）。

## 4. 保守性・互換性 (Maintainability & Compatibility)

### 4.1. ブラウザ互換性
- Google Chrome 最新版（および過去数バージョン）で動作すること。
- 将来的にはEdge等のChromium系ブラウザへの対応も考慮する。

### 4.2. YouTube UI追従
- YouTube側のDOM構造変更（クラス名の難読化変更など）に対して、比較的容易に追従・修正できるアーキテクチャ（メンテナンス性の高いセレクタ管理など）を採用すること。
