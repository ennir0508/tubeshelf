# 機能要件定義書 (Functional Requirements Definition)

## 1. カテゴリ管理機能

### 1.1. カテゴリ作成 (FR-01)
- ユーザーは任意の名前で新しいカテゴリ（フォルダ）を作成できる。
- カテゴリ名には汎用的な文字が使用可能であること。

### 1.2. カテゴリ編集・削除 (FR-02)
- 作成済みのカテゴリ名を変更できる。
- カテゴリを削除できる。
    - カテゴリ削除時、内部に含まれるチャンネルは「未分類」に戻るか、親カテゴリに移動するかを選択・定義する（デフォルトは未分類に戻る）。

### 1.3. 階層構造 (FR-03)
- カテゴリは入れ子構造（ネスト）を持つことができる。
- 親カテゴリの中に子カテゴリを作成可能。

## 2. 登録チャンネル整理機能

### 2.1. 所属変更 (FR-04)
- 登録チャンネルを任意のカテゴリに割り当てることができる。
- 1つのチャンネルは原則1つのカテゴリに属する（またはタグのように複数所属を許容するか検討要。本書ではまず「フォルダ形式（1対1）」を基本とする）。

### 2.2. ドラッグ＆ドロップ操作 (FR-05)
- UI上でチャンネルアイコンをドラッグし、カテゴリフォルダにドロップすることで移動できる。

### 2.3. 自動分類 (FR-06) [Nice to have]
- チャンネル名やキーワードに基づく自動振り分け機能（将来的な拡張）。

## 3. UI表示・統合機能

### 3.1. サイドバー注入 (FR-07)
- YouTube左側のナビゲーションドロワー（サイドバー）内に、本拡張機能専用のセクションを表示する。
- デフォルトの登録チャンネルリストを置換、または並列して表示する。
- YouTube本来のデザイン（Look & Feel）に馴染むデザインとする。

### 3.2. 開閉状態の保持 (FR-08)
- カテゴリの展開・折りたたみ状態は保存され、次回アクセス時も維持される。

### 3.3. 未分類チャンネルの表示 (FR-09)
- カテゴリに属していないチャンネルも一覧でき、迷子にならないようにする。

## 4. 動画視聴支援機能

### 4.1. カテゴリ絞り込み表示 (FR-10)
- カテゴリをクリックすると、そのカテゴリに含まれるチャンネルの動画のみをフィルタリングした「フィード」を表示する機能。
- または、そのカテゴリに属するチャンネルリストを展開表示し、個別のチャンネルへ遷移しやすくする。

### 4.2. 新着バッジ (FR-11)
- カテゴリ内のチャンネルに新着動画がある場合、カテゴリ自体に未読バッジやドットを表示し、更新があることを通知する。
