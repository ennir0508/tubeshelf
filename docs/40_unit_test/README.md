# 単体テスト (Unit Test)

このディレクトリでは、単体テストの方針、計画、および実施結果に関する情報を管理します。
各コンポーネントが孤立した状態で正しく動作することを保証します。

## ドキュメント構成

本ディレクトリには、以下のドキュメントを配置します。

| ドキュメント         | ファイル名                                       | 内容概要                                                                                                         |
| :------------------- | :----------------------------------------------- | :--------------------------------------------------------------------------------------------------------------- |
| **単体テスト方針**   | [`test_policy.md`](./test_policy.md)             | テスト対象の範囲、使用するテストフレームワーク（Vitest/Jest）、モックの方針などを定めます。                      |
| **カバレッジ基準**   | [`coverage_criteria.md`](./coverage_criteria.md) | 目標とするコードカバレッジ率（ステートメント、ブランチ等）と、その運用ルールを記述します。                       |
| **テストケース一覧** | [`test_cases.md`](./test_cases.md)               | 主要なテストケースのカテゴリ分けや、特筆すべきテストパターンを管理します（詳細はコード内のspecファイルに記述）。 |
