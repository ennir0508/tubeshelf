# 開発 (Development)

このディレクトリでは、実装工程に関するルール、ガイドライン、およびコードそのものに関する説明を管理します。
品質の均一化と保守性の向上を目的とした情報を集約します。

## ドキュメント構成

本ディレクトリには、以下のドキュメントを配置します。

| ドキュメント             | ファイル名                                           | 内容概要                                                                                                   |
| :----------------------- | :--------------------------------------------------- | :--------------------------------------------------------------------------------------------------------- |
| **コーディング規約**     | [`coding_convention.md`](./coding_convention.md)     | プログラミング言語（TypeScript/React）ごとの命名規則、フォーマットルール、使用禁止事項などを定めます。     |
| **ディレクトリ構成図**   | [`directory_structure.md`](./directory_structure.md) | `src`ディレクトリ以下の詳細な構成と、各ディレクトリの責務を説明します。                                    |
| **ブランチ運用ルール**   | [`branch_strategy.md`](./branch_strategy.md)         | Gitのブランチ戦略（GitHub Flow等）、コミットメッセージのフォーマット、プルリクエストのルールを定義します。 |
| **環境構築・ビルド手順** | [`build_setup.md`](./build_setup.md)                 | 開発環境のセットアップ手順、依存ライブラリの管理、ビルドコマンドの使い方を記述します。                     |
